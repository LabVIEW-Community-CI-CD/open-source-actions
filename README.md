
## 📚 Governance Framework (v2025.05)

The NI Open-Source Program is guided by a transparent, community-influenced governance model.  
As of **May 2025**, the following documents define how we prioritize, evaluate, and support open-source IP:

| Document | Purpose |
|----------|---------|
| [`PROGRAM-GUIDE.md`](./docs/governance/PROGRAM-GUIDE.md) | Overview of program goals, lifecycle stages, and repo evaluation |
| [`PRIORITY-SCORE.md`](./docs/governance/PRIORITY-SCORE.md) | Defines the four-factor model for R&D resource allocation |
| [`STEERCO-GUIDELINES.md`](./docs/governance/STEERCO-GUIDELINES.md) | Clarifies Steering Committee responsibilities and boundaries |
| [`CONTRIBUTOR-RECOGNITION.md`](./docs/governance/CONTRIBUTOR-RECOGNITION.md) | Badge and credit system based on public GitHub activity |
| [`ENGAGEMENT-GUIDE.md`](./docs/governance/ENGAGEMENT-GUIDE.md) | Contributor expectations for GitHub Issues, PRs, and testing |
| [`MEETING-POLICY.md`](./docs/governance/MEETING-POLICY.md) | Cadence, agenda, and documentation standards for program meetings |
| [`GOVERNANCE-CHANGELOG.md`](./docs/governance/GOVERNANCE-CHANGELOG.md) | Time-stamped history of governance decisions and policy changes |
