# GOVERNANCE-SCOPE.md

> 🧭 This file defines the official boundaries of the NI Open-Source Governance Framework — what it controls, what it excludes, and where contributors or reviewers should escalate for clarification.

---

## ✅ In-Scope (Governed)

The following topics are formally governed, enforced, and versioned within this framework:

- Contributor recognition (badges, opt-in, eligibility)
- Issue templates, PR structure, and manual test enforcement
- Repository scoring and prioritization
- SteerCo roles, meeting rules, and participation expectations
- GitHub label-based recognition or scoring triggers
- Assistant behavior within protected `.md` files
- Change control of governance files
- Traceability of overrides (`#manual-edit`, `#manual-override`)
- Governance versioning and audit trail (`GOVERNANCE-VERSION`, milestones)

---

## ❌ Out-of-Scope (Currently Uncovered)

The following are **not** currently governed by this framework:

- Internal team prioritization outside GitHub
- Legal compliance review of third-party contributions
- Contributor Code of Conduct enforcement *(in progress)*
- Paid recognition, sponsorships, or monetized badges
- Project release decisions or roadmap commitments by NI R&D
- External IP licensing for repositories outside this program

---

## 🔄 Escalation for Ambiguities

If a topic falls outside this list — or seems only partially governed — escalate via:

- GitHub Discussions: [ni/open-source/discussions](https://github.com/ni/open-source/discussions)
- Email: `sergio.velderrain@emerson.com`
- GitHub issue tagged `governance-appeal`

See `AI-GOVERNANCE-CONTRACT.md` and `OVERRIDE-GUIDE.md` for enforcement pathways.
