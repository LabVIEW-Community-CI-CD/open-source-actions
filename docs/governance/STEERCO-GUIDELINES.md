# STEERCO-GUIDELINES.md

> 🧭 Defines the responsibilities, boundaries, participation model, and recognition criteria for Steering Committee (SteerCo) members within the NI Open-Source Program.

---

## Role Definition

SteerCo members are volunteer technical leaders who:

- Represent real-world usage, integration, and validation of NI open-source IP
- Provide roadmap input, surface issues, and guide contributor onboarding
- Are publicly recognized for their engagement

Note: **SteerCo membership increases project eligibility** but does not authorize decisions on project prioritization or launches. See `PRIORITY-SCORE.md`.

---

## Responsibilities and Boundaries

| Area                 | SteerCo Role                                              |
|----------------------|-----------------------------------------------------------|
| 🧪 Technical Guidance | Review PRs, define usage patterns, surface bugs           |
| 🧭 Evaluation Input   | Assess interest and practicality (non-binding)            |
| 🔁 Community Support  | Encourage new contributors, share test feedback           |
| ❌ Governance Changes | Cannot override NI policy or launch decisions             |
| ❌ Roadmap Control    | Cannot assign priority labels (P0–P2)                      |

---

## Participation Model

| Element          | Expectation |
|------------------|-------------|
| Weekly Time      | ~2 hours/week per repo or category |
| Activity Channels| GitHub PRs, Issues, Discussions     |
| Collaboration    | Coordinate with other SteerCo members, NI stakeholders |
| Independence     | Contribute as individuals, not company reps |
| Visibility       | All participation must be GitHub-visible and auditable |

---

## Meeting Rules

| Rule        | Description |
|-------------|-------------|
| Scheduling  | Meetings occur **only when requested** by SteerCo or NI — no standing cadence |
| Agendas     | Required 24 hours in advance for any meeting (ad hoc or scheduled) |
| Separation  | Governance and technical IP meetings must be scoped and documented separately |
| Notes       | Key outcomes must be logged in GitHub Issues or Discussion threads |

---

## Membership Lifecycle

### To Join:
- Express interest via [GitHub Discussion thread](https://github.com/ni/open-source/discussions/22)
- Undergo approval via private vote from the Open Source Core Team
- Commit to 2 hours/week of GitHub activity

### To Step Down:
- Notify the Program Manager via GitHub (preferred) or Discord
- Change of status must be logged publicly for traceability

---

## Recognition and Eligibility

SteerCo members:
- Are eligible for public recognition via badges (see `CONTRIBUTOR-RECOGNITION.md`)
- Increase a repo’s visibility and eligibility for R&D investment (see `PRIORITY-SCORE.md`)
- May influence NI product decisions **only through transparent, documented engagement**
