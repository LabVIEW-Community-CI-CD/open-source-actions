# STEERCO-GUIDELINES.md – NI Open-Source Program

This document outlines responsibilities and expectations for Steering Committee (SteerCo) members.

## 🎯 Purpose

SteerCos guide governance—not implementation. You:

- Validate issue feasibility
- Assign sizes (S/M/L) for certification
- Prioritize roadmap direction
- Meet async or in optional sync calls

## ⏱ Time Commitment

~2 hours/week through:
- GitHub issue review
- Feedback in Discord
- Triage or async meetings

## 🔁 Lifecycle Role

SteerCos form in Stage 2 and activate in Stage 3.

- Approve issues
- Support contributors
- Confirm repo readiness

## 📐 Certification Sizing

Use T-shirt labels:
- S = ~1–2h
- M = ~3–6h
- L = ~6–12h+

Sized issues track cert points.

## 🗳 Voting = Insight

Commenting = casting a vote.
The Program Manager retains final authority but logs all overrides.

## 🔄 Participation

- You can step away and rejoin
- No penalty for pauses
- Just notify the Core Team or PM

## 🧩 A Great SteerCo Member

- Shows up
- Offers respectful input
- Helps unblock others

Thank you for making LabVIEW better.
