# NI Open-Source Program Overview

## Executive Summary

This document provides an in-depth overview of the NI Open-Source Program. It outlines how NI is leveraging open-source collaboration to sustainably maintain and modernize legacy IP, incentivize community participation, and enable professional growth through certification-aligned contribution.

This program is built on a flexible governance model and integrates seamlessly with LabVIEW's recertification process. It does **not** provide initial certification — only **recertification credits** to eligible individuals.

**Key Highlights for Executives:**

- Designed for strategic alignment between NI-owned IP and external innovation.
- Reduces engineering backlog by transforming contributors into distributed roadmap accelerators.
- Directly supports LabVIEW recertification (for eligible holders) through traceable open-source contributions.
- Ensures governance accountability through BDFL control, structured SteerCos, and metadata-backed traceability.

## Strategic Objectives

- Leverage community expertise to evolve high-value, NI-governed IP.
- Reduce internal engineering overhead while increasing long-term maintenance capacity.
- Reinforce NI’s leadership in the LabVIEW ecosystem through accessible recertification-by-contribution.
- Support roadmap-aligned feature expansion with minimal overhead by empowering SteerCos.

---

## Governance Structure

### Program Manager (BDFL)
- Acts as the final decision-maker across all SteerCos and project tracks.
- Ensures technical alignment with internal NI teams and product line owners.
- Logs and justifies overrides of SteerCo votes to preserve transparency.

### Technical Steering Committees (SteerCos)
- Application-based volunteer groups aligned to repositories or features.
- Guide direction, feasibility, and prioritization of issues safe for public contribution.
- Review issues for certification value and T-shirt sizing.

### Organizational Steering Committee (OSC)
- Handles risk, IP posture, licensing, and organizational policy alignment.
- Comprised of NI stakeholders and governance support roles.

## Participation Framework

- Designed for contributors with limited weekly time (target: ~2 hours/week).
- SteerCo members provide insight, not implementation, though both are welcomed.
- Contributors claim GitHub issues aligned to their skill level and available time.
- All contribution and engagement data is logged on GitHub and integrated into project KPIs.

---

## Recertification Integration

- This program supports **LabVIEW recertification only** — not initial certifications.
- Only eligible individuals (e.g., holders of CLA, CLD, etc.) may receive recertification credit.
- Contributors earn points by completing SteerCo-approved GitHub issues.
- Each issue is labeled with a **T-shirt size** (S/M/L) to represent effort level.
- Points are traceable via GitHub metadata and exported to NI’s internal systems.

This alignment reduces manual work for the Certification team and allows for scalable, automated validation.

---

## Contribution Lifecycle

1. **Issue Proposal** – Submitted by NI, SteerCo, or the community.
2. **Feasibility Review** – SteerCo confirms the issue is safe for external contribution.
3. **Sizing** – The issue is labeled with a T-shirt size (S/M/L).
4. **Approval** – The Program Manager validates and opens the issue for contribution.
5. **Implementation** – Contributor claims the issue and submits a PR.
6. **Review + Merge** – Maintainers review and merge code.
7. **Certification Credit (if eligible)** – Logged and processed for recertification.

---

## Operational Transparency

- All meetings are transcribed; key decisions are time-stamped and documented.
- Program Manager maintains override logs when SteerCo input is superseded.
- Public recognition is opt-in; no participation is ever made public without consent.
- All interactions occur via GitHub and Discord — no proprietary systems required.

## FAQ

**Q1: Who can contribute?**  
**A:** Anyone. Certification eligibility only applies if you’re seeking recertification points.

**Q2: Can NI internal teams participate?**  
**A:** Yes, NI employees and external contributors collaborate equally under the same process.

**Q3: Why use a BDFL model?**  
**A:** It minimizes decision-making latency and ensures program alignment without requiring full-time community governance.

**Q4: What safeguards exist for contributor experience?**  
**A:** Issue capacity is scaled based on available SteerCo input. SteerCo members signal availability to prevent overcommitment.

**Q5: Is this model intended to evolve?**  
**A:** Yes. As contributor engagement scales, NI may pilot new delegation models and governance policies to extend sustainability.
