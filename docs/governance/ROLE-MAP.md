# ROLE-MAP.md

> 🧑‍💼 Maps all current and planned governance roles to their defined authorities, owned files, and decision boundaries.

---

## Role Definitions and Scope

| Role                    | Description |
|-------------------------|-------------|
| **Program Manager & Architect** | Owns governance authority, enforcement rules, scoring logic, and badge issuance; defines system evolution |
| **Steering Committee (SteerCo)** | Provides technical guidance and contributor evaluation; cannot assign priority or make launch decisions |
| **Maintainer** *(planned)* | Day-to-day PR review, test merge authority, and roadmap collaboration; eligible for maintainer badge |
| **Core Team** *(planned)* | Backup enforcement, dispute resolution, governance proposal review |
| **Certification Team** *(planned)* | Manages certification eligibility, test validation, and contributor scoring model inputs |

## Role Authority Map

| Role                    | Can Approve | Cannot Approve |
|-------------------------|-------------|----------------|
| **Program Manager**     | Badges, scoring, SteerCo onboarding, milestone approval | CLA exceptions without Legal input |
| **SteerCo**             | Contributor recommendations, interest metrics | Roadmap access, project scoring |
| **Maintainer** *(planned)* | Test validation, PR review | Badge issuance, milestone sign-off |
| **Core Team** *(planned)* | Override review, proposal triage | System-wide enforcement changes |
